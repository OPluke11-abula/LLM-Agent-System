# 🤖 AI Agent 每次工作結束自我檢核與規範 (AGENT.md)

本文件定義了 AI Agent 在每次工作结束時必須嚴格遵循的「自我檢核與作業規範」。

---

## 📋 每次工作結束核心作業規則

每當您完成一項任務或一階段的開發工作，在結束該 Turn 之前，請務必依序執行以下 5 項核心步驟：

### 1. 🔍 檢查並修復是否有冗餘程式碼與 BUG
- **冗餘程式碼清理**：刪除測試殘留、除錯用 Print、未使用的 import、臨時註解或重複的輔助函式。
- **BUG 排查與修正**：檢視變更的程式碼是否含有型別錯誤、資源未釋放、例外處理未捕獲或邊界條件處理不當等問題。

### 2. 🏗️ 檢查框架與架構是否正確
- **職責分離**：確認變更是否遵循 `core/`、`skills/` 與 `memory/` 之間的邊界，不將 UI、HTTP、拓撲或 PAP 協定邏輯混入核心引擎。
- **Contract-First 理念**：確認 Python 的 Pydantic 工具定義與 `.agent/skills/` 的 Markdown 合約完全一致。

### 3. 🧠 自主判斷是否更新 `.agent/` 目錄內容
- **身份合約 (`.agent/agent.md`)**：如果擴充了新的核心能力、工具或調整了身分定位（如 Programmer / 程序員），自主判斷並更新之。
- **任務隊列 (`.agent/agent_tasks.md`)**：根據目前的開發進度，更新任務清單的狀態（如將 `[ ]` 標記為 `[x]` 或 `[~]` 等）。

### 4. 📝 更新並確認 `README.md` (中文與英文分開)
- **繁體中文與 English 獨立更新**：如果新增了功能、指令或修改了架構，必須同步更新 `README.md` 的 [English](#english) 與 [繁體中文](#繁體中文) 段落。
- **結構清晰**：確保中英文內容結構分開，並提供清晰明瞭的對應說明。

### 5. 🚀 Git 提交與推送 (Commit/Push)
- **預檢測試**：在 Commit 前，先執行 `verify.cmd` 或 pytest 測試套件，確保 100% 綠燈。
- **規範提交**：執行 `git add .`、符合規範的 `git commit` 以及 `git push` 到遠端儲存庫。
